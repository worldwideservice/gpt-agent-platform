# Архитектура Kwid - Полная структура

> Документ собран на основе анализа сервиса Kwid через Playwright  
> Дата: 2025-01-XX  
> URL: https://aai.widgets.wearekwid.com/manage/1000373-worldwideservices

## Глобальная структура навигации

### Header (верхняя панель)
- **Глобальный поиск** - поисковая строка (placeholder: "Глобальный поиск")
- **Дата окончания подписки** - ссылка на `/pricing` (например: "30.10.2025")
- **Уведомления** - кнопка с счетчиком непрочитанных (например: "22")
  - При клике открывается модальное окно с уведомлениями
  - Фильтры уведомлений (например: "Ответов ИИ")
  - Список уведомлений
- **Меню пользователя** - аватар пользователя (выпадающее меню с действиями)
  - Имя пользователя (например: "Admin")
  - Переключатели темы:
    - **"Включить светлый режим"**
    - **"Включить темный режим"**
    - **"Включить системный режим"**
  - Кнопка **"Выйти"** (logout)

### Sidebar (боковое меню)

#### Структура меню:
```
GPT Агент (логотип)
├── Инфопанель (/manage/[tenantId])
├── Агенты ИИ
│   ├── Агенты ИИ (/ai-agents)
│   └── Тестовый чат (/test-chat)
├── База знаний
│   ├── Категории (/knowledge-categories)
│   └── Статьи (/knowledge-items)
├── Поддержка
│   └── Начало работы (внешняя ссылка: /docs/ru/start-here/getting-started/)
├── Аккаунт
│   ├── Настройки аккаунта (/account-settings)
│   └── Тарифные планы (/pricing)
└── Что нового
    ├── Смотреть на Facebook (внешняя ссылка)
    └── Смотреть в Instagram (внешняя ссылка)
```

## Страницы

### 1. Dashboard / Инфопанель

**URL:** `/manage/[tenantId]`

#### Компоненты:

**Виджеты метрик (4 карточки):**
1. **Ответы ИИ за этот месяц**
   - Значение: число (например: "924")
   - Изменение: процент к прошлому месяцу (например: "-94.1% к прошлому месяцу")
   - Иконка тренда

2. **Ответы ИИ за последние 7 дней**
   - Значение: число (например: "1574")
   - Описание: "Последние 7 дней"

3. **Ответы ИИ сегодня**
   - Значение: число (например: "0")
   - Изменение: процент к вчерашнему дню (например: "-100% к вчерашнему дню")

4. **Агенты**
   - Значение: число (например: "1")
   - Описание: "Всего агентов"

**Графики:**
- График "Ответы ИИ за этот месяц" (заголовок H3)
- График "Ответы ИИ за день" (заголовок H3)

#### Breadcrumbs:
- Нет (главная страница)

### 2. AI Agents List / Список агентов

**URL:** `/manage/[tenantId]/ai-agents`

#### Компоненты:

**Заголовок страницы:**
- H1: "Агенты ИИ"
- Breadcrumbs: "Агенты ИИ" → "Список"

1. Checkbox для массовых действий

- **Создать** (в заголовке)

## API Endpoints (предположительно)

На основе структуры URL можно предположить следующие endpoints:

- `GET /api/agents` - список агентов
- `POST /api/agents` - создание агента
- `GET /api/agents/:id` - получение агента
- `PUT /api/agents/:id` - обновление агента
- `DELETE /api/agents/:id` - удаление агента
- `POST /api/agents/:id/copy` - копирование агента

- `GET /api/knowledge-items` - список статей
- `POST /api/knowledge-items` - создание статьи
- `GET /api/knowledge-items/:id` - получение статьи
- `PUT /api/knowledge-items/:id` - обновление статьи
- `DELETE /api/knowledge-items/:id` - удаление статьи

- `GET /api/knowledge-categories` - список категорий
- `POST /api/knowledge-categories` - создание категории
- `GET /api/knowledge-categories/:id` - получение категории
- `PUT /api/knowledge-categories/:id` - обновление категории
- `DELETE /api/knowledge-categories/:id` - удаление категории

- `GET /api/dashboard/stats` - статистика для дашборда
- `GET /api/crm/sync` - синхронизация с CRM
