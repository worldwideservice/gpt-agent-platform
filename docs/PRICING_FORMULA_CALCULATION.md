# Расчет формулы цены на основе KWID

## Данные с KWID:

**Scale:**
- 1,000: $45
- 5,000: $205
- 15,000: $578
- 20,000: $760

**Max:**
- 1,000: $90
- 5,000: $350
- 15,000: $973
- 20,000: $1,280

## Расчет формулы для Scale:

Используем метод наименьших квадратов для линейной регрессии:
y = a + b*x

Где:
- y = цена
- x = количество ответов
- a = базовая цена
- b = цена за ответ

Из данных:
- 45 = a + 1000*b
- 205 = a + 5000*b
- 578 = a + 15000*b
- 760 = a + 20000*b

Решая систему:
b = (760 - 45) / (20000 - 1000) = 715 / 19000 = 0.0376
a = 45 - 1000*0.0376 = 45 - 37.6 = 7.4

Но это не точно. Попробуем другой подход - используем среднее:
b ≈ (760 - 45) / (20000 - 1000) ≈ 0.0376
a ≈ 45 - 1000*0.0376 ≈ 7.4

Но для 5,000: 7.4 + 5000*0.0376 = 7.4 + 188 = 195.4 (должно быть 205)

Попробуем более точную формулу:
Для Scale: базовая ≈ $8, множитель ≈ $0.0376

## Расчет для Max:

- 90 = a + 1000*b
- 350 = a + 5000*b
- 973 = a + 15000*b
- 1280 = a + 20000*b

b ≈ (1280 - 90) / (20000 - 1000) ≈ 1190 / 19000 ≈ 0.0626
a ≈ 90 - 1000*0.0626 ≈ 27.4

Проверка для 5,000: 27.4 + 5000*0.0626 = 27.4 + 313 = 340.4 (должно быть 350)

Более точная формула для Max:
a ≈ $30, b ≈ $0.06
